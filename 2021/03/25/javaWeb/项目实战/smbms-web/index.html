    3.5.2. 编写query方法

  • 获得前端请求req中的参数

  • 初始化参数

    • userService,service层依赖
    • userList用户列表
    • queryUserRole后台用户角色参数,默认设置0,即为查询所有
    • queryUserName后台用户名参数,默认设置为空,即为查询所有
    • currentPageNo设置初始页号为1
    • pageSize默认设置每页有五条数据
    • 设置queryUserName等于req传递的参数queryname
    • 设置queryUserRole等于req传递的参数queryUserRole字符的int形式
    • 设置pageIndex等于req传递的参数pageIndex字符的int形式

  • 调用PageSuport工具类获取页面参数

  • 控制页面的首尾页

    • 如果页面小于1就显示第一页
    • 如果页面大于总页数就显示最后一页

  • 获取用户列表展示

3.7. 实现UserServlet的用户信息添加,查看,删除,修改

3.7.1. ajax异步实现数据删除

  • 发送请求path+”/jsp/user.do?method=deluser&uid=”+userid
  • key1=method value1=deluser
  • key2=uid value2=userid(前端获得的字段id)

    3.7.2. ajax异步实现查找角色列表

  • 发送请求path+”/jsp/user.do?method=getrolelist”
  • key=method value=getrolelist

    3.7.3. ajax异步判断用户是否已经存在

  • 前端发送请求path+”/jsp/user.do?method=ucexist&userCode=”+userCode
  • key1=method value1=ucexist
  • key2=methdo value2=userCode(前端传递的用户编号)

3.7.4. 代码复用

  • 前端请求method=add,调用添加用户的方法add(req, resp);
  • 前端请求method=getrolelist,调用获得角色列表的方法getrolelist(req, resp);
  • 前端请求method=ucexist,调用判断用户是否存在的方法userCodeExist(req, resp);
  • 前端请求method=deluser,调用删除的方法delUser(req, resp);
  • 前端请求method=view,调用跳转查看数据页面的方法getUserById(req, resp, “userview.jsp”);
  • 前端请求method=modify,调用跳转更新页面的方法getUserById(req, resp,”usermodify.jsp”);
  • 前端请求method=modifyexe,调用提交更新数据表单的方法modifyUser(req,resp);
